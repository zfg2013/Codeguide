# File Naming and Storage

Owner: Data steward  
Last reviewed: 2026-05-06  
Status: Example guide

## Purpose

Make files understandable without relying on personal folders or private notes.

## Naming Pattern

Use names that include project, sample or cohort, data type, processing stage, and date when useful.

```text
PROJECT_COHORT_DATATYPE_STAGE_YYYYMMDD.ext
```

Example:

```text
alzheimers_pilot_rnaseq_counts_raw_20260506.tsv
```

## Storage Tiers

| Tier | Use | Example |
| --- | --- | --- |
| Raw | Original data from instrument, core, or collaborator | FASTQ, raw images, intake sheets |
| Processed | Generated by documented workflow | Count matrices, QC tables |
| Results | Figures, tables, reports | Differential expression results |
| Archive | Inactive but retained projects | Frozen release folders |

## Rules

- Do not edit raw data in place.
- Store scripts and notebooks near the project repository, not only next to data.
- Include a `README.md` in each project data folder.
- Record controlled-access restrictions at the top of the project README.

## Project Handoff Checklist

- Main project folder identified.
- Raw, processed, and results folders documented.
- Active workflows linked from this knowledge base.
- Open questions and known data issues listed.
- New owner has access before the old owner leaves.
