# Variant QC Summary

Owner: Computational lead  
Last reviewed: 2026-05-06  
Status: Example workflow

## Purpose

Summarize variant call quality before downstream association or prioritization analysis.

## Inputs

| Input | Location | Notes |
| --- | --- | --- |
| Joint-called VCF | `shared/projects/example_variants/vcf/cohort.joint.vcf.gz` | Indexed with tabix |
| Pedigree or sample metadata | `shared/projects/example_variants/metadata/samples.tsv` | Includes sample ID and phenotype group |
| Reference genome | `shared/references/GRCh38/GRCh38.fa` | Must match alignment reference |

## Steps

```bash
bcftools stats \
  --samples - \
  shared/projects/example_variants/vcf/cohort.joint.vcf.gz \
  > results/variant_qc/bcftools_stats.txt
```

```bash
plot-vcfstats \
  -p results/variant_qc/plots \
  results/variant_qc/bcftools_stats.txt
```

## Review Checklist

- Confirm expected sample count.
- Check transition/transversion ratio.
- Review missingness by sample and variant.
- Confirm chromosome naming matches downstream tools.
- Flag any sample with unusually high heterozygosity or missingness.

## Outputs

| Output | Meaning |
| --- | --- |
| `bcftools_stats.txt` | Summary statistics from the VCF |
| `plots/` | Standard plots generated by `plot-vcfstats` |
| `qc_notes.md` | Manual review comments and exclusions |

## Escalation

If sample-level QC suggests contamination, sample swap, or sex mismatch, pause downstream analysis and notify the project PI and data steward.
