# First Week Checklist

Owner: Lab manager  
Last reviewed: 2026-05-06

## Before Arrival

- Confirm institutional account is active.
- Add the new member to lab email lists and shared calendars.
- Prepare workstation or remote access instructions.
- Identify the person's initial mentor for day-to-day questions.

## Day 1

- Review lab communication norms and meeting schedule.
- Confirm access to the shared drive, code repositories, and compute cluster.
- Show where current protocols and analysis workflows are documented.
- Review data handling expectations for human subject or controlled-access data.

## First Week

- Run the local documentation preview.
- Clone one existing analysis repository in read-only mode.
- Complete a small documentation edit through a pull request.
- Meet with the data steward to review storage, backups, and naming conventions.

## Common Access Requests

| System | Request Path | Owner |
| --- | --- | --- |
| Compute cluster | Institutional IT ticket | Computational lead |
| Shared drive | Lab manager request | Lab manager |
| GitHub organization | Repository admin invite | Computational lead |
| Freezer inventory | Lab manager onboarding form | Lab manager |

## Exit Criteria

The person can find protocols, identify where project data live, preview this site locally, and knows who to ask before moving or deleting shared files.
