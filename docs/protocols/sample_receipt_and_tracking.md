# Sample Receipt and Tracking

Owner: Lab manager  
Last reviewed: 2026-05-06  
Status: Example protocol

## Purpose

Ensure incoming samples are logged, labeled, stored, and linked to project metadata before any processing begins.

## Scope

Applies to human, animal, and cell-line samples received for active lab projects. Follow project-specific compliance requirements when controlled-access or identifiable data are involved.

## Materials

- Approved sample intake form.
- Preprinted freezer labels or lab-approved label printer.
- Freezer inventory access.
- Project metadata folder.

## Procedure

1. Confirm the shipment matches the intake form.
2. Assign or confirm the lab sample ID.
3. Record receipt date, condition, volume, and storage location.
4. Label tubes before moving them to long-term storage.
5. Upload the intake form to the project metadata folder.
6. Notify the project owner that samples are available.

## Quality Checks

- Every tube has a lab sample ID.
- Inventory location includes freezer, rack, box, row, and column.
- Metadata file contains the same sample IDs as the physical labels.
- Deviations are documented in the project notes.

## Stop Conditions

Do not process samples if labels are missing, the intake form is incomplete, or the shipment appears compromised.
