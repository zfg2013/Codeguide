# Philosophy

This knowledge base exists to reduce avoidable dependency on individual memory.

## Working Principles

- Boring infrastructure is a feature.
- Markdown is the source of truth.
- A good page is one that another lab member can edit.
- The most useful documentation records context, not just commands.
- Troubleshooting notes are lab knowledge and should be searchable.
- Raw data should be protected from accidental edits.
- Every important workflow should explain inputs, outputs, ownership, and limitations.

## What To Avoid

- Custom web apps for simple documentation.
- Pages that only say "ask Alice" or "see old Slack thread."
- Unversioned screenshots of commands when copyable text would work.
- Hidden personal folders as the only record of an analysis.
